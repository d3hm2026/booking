# قاعدة بيانات نظام إدارة الاستراحات والشاليهات

## خطوات الإعداد على Supabase

### 1. إنشاء مشروع جديد
- روح إلى [supabase.com](https://supabase.com) وسجّل دخول (أو أنشئ حساب)
- أنشئ مشروع جديد (New Project)، اختر باسوورد قوي لقاعدة البيانات واحفظه
- انتظر دقيقة لين يجهز المشروع

### 2. تشغيل ملفات الـ Migrations
روح إلى **SQL Editor** في لوحة تحكم Supabase، وشغّل الملفات **بالترتيب التالي تماماً** (كل ملف في استعلام جديد):

1. `001_initial_schema.sql` — إنشاء كل الجداول
2. `002_booking_conflict_check.sql` — فحص تعارض الحجوزات تلقائياً
3. `003_row_level_security.sql` — تفعيل الحماية (RLS)

### 3. (اختياري) إضافة بيانات تجريبية
لو تبي تتأكد إن كل شيء شغال، شغّل ملف:
- `seed_test_data.sql`

بيانات الدخول التجريبية بعد تشغيله:
| الدور | رقم الجوال | الرمز السري |
|---|---|---|
| أدمن | 0500000000 | 1234 |
| مالك | 0500000001 | 5678 |
| عامل تنظيف | 0500000002 | 9999 |

⚠️ **مهم:** هذا الملف لبيانات تجريبية فقط. لا تشغّله على قاعدة بيانات حقيقية فيها بيانات فعلية، وامسح هذه البيانات التجريبية قبل الاستخدام الفعلي.

### 4. الحصول على مفاتيح الربط
من **Project Settings → API**، احفظ:
- `Project URL`
- `anon public key`
- `service_role key` ⚠️ (سري جداً - لا تشاركه أو تضعه في كود يصل للمتصفح)

هذه المفاتيح ستُستخدم في الخطوة التالية (إعداد مشروع Next.js).

### 5. إعداد تخزين الصور (للمرحلة القادمة - مهام التنظيف)
من **Storage** في لوحة Supabase، أنشئ bucket باسم `cleaning-photos` وحدد صلاحياته بحيث:
- القراءة: فقط من السيرفر (private)
- الرفع: فقط من السيرفر (private)

سنغطي ربط هذا الجزء بالتفصيل عند بناء شاشة عامل التنظيف.

## ملخص بنية الجداول

| الجدول | الوصف |
|---|---|
| `owners` | الملاك |
| `users` | حسابات الدخول (admin / owner / cleaner) |
| `units` | الوحدات (الاستراحات/الشاليهات) |
| `unit_daily_prices` | السعر اليومي لكل وحدة |
| `unit_blocks` | فترات حجب الوحدة (صيانة، استخدام شخصي) |
| `bookings` | الحجوزات |
| `payments` | الدفعات (تحويل بنكي) |
| `cleaning_tasks` | مهام التنظيف |
| `cleaning_photos` | صور ما بعد التنظيف |
| `booking_logs` | سجل تتبع تعديلات الحجز |

## ميزات الحماية المدمجة في قاعدة البيانات

- **منع تعارض الحجوزات تلقائياً**: لا يمكن إدخال حجزين متداخلين على نفس الوحدة (Migration 002)
- **منع الحجز في فترة محجوبة**: لا يمكن حجز وحدة في فترة صيانة مسجلة
- **سجل تلقائي**: كل حجز جديد يُسجَّل تلقائياً في `booking_logs`
- **RLS مفعّلة بالكامل**: لا يمكن لأي طرف الوصول لقاعدة البيانات إلا من خلال كود السيرفر (Next.js) المستخدم لـ `service_role key`
